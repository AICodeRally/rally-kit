# Demo — Generate Presentation Script

Create a 2-minute demo script the team can use for their presentation.

## Instructions

1. Read `DOMAIN.md` — understand the business
2. Read `.rally-progress` — know what pages exist
3. Read `.team-name` and `.team-members` — personalize the script
4. Read `.rally-port` — include the correct localhost URL

### Generate the Demo Script

Write `DEMO_SCRIPT.md` with this structure:

```markdown
# [Team Name] — Demo Script

## The Pitch (2 minutes)

### 1. The Problem (15 seconds)
"[One sentence about the pain point this solves]"

### 2. Our Solution (15 seconds)
"We built [app name] — [one sentence about what it does]"

### 3. Live Demo (60 seconds)
Walk through each page:
- **[Page 1]**: Show [key feature]. Point out [impressive detail].
- **[Page 2]**: Show [key feature]. Point out [impressive detail].
- **[Page 3]**: Show [key feature]. Point out [impressive detail].

### 4. The Vision (15 seconds)
"If we had more time, we'd add [realistic next feature]. Our vision is [big picture]."

### 5. Close (15 seconds)
"We're [Team Name]. We built this in 3 hours with AI. Questions?"

## Tips
- Make eye contact with judges, not the screen
- Point at specific things on screen as you demo
- If something breaks during demo, laugh it off: "Live demos, right?"
- Speak slowly — you have more time than you think
```

### After saving:

Tell them:
> "Your demo script is saved! Pull it up on your phone or another screen during the presentation. Want to practice? Pretend I'm the judges — walk me through it and I'll give feedback."

If a Presenter role was assigned (check `.team-roles`), address them:
> "[Presenter name], this is your moment — ready to practice?"
